# 网络之 HTTP概述


Q1：什么是HTTP(超文本传输协议)
A：
  - 首先是 协议，意味着它不是实体，而只是一种约定和规范；
  - 其次是 传输协议，意味着 HTTP主要是对 规定/约束了 `如何把某个资源从 A 运送到 B`
  - 最后是超文本，表明了 HTTP传输的数据不是二进制包，而是有意义的数据

  - PS1：HTTP是一个双向协议，A/B 之间可以互相主动发起传输，而不只是单向传输
  - PS2：A/B 传输之间允许中间人/代理存在，即  A <--> [C] <--> [X] <--> B
------

Q2：从传输的角度看，HTTP的涉及主体，简单有哪些
A：
  - user Agent：用户代理，如 浏览器/爬虫等
  - web Server:  服务端
  - CDN: 内容分发网络，用于 资源内容缓存/安全防护/资源请求负载均衡等
-----

Q3：从网络层级的角度看，HTTP大致涉及到了哪些 其他网络协议或者技术
A：
  - 网络层协议- IP：用于实现 主机寻址/路由分发 等功能；
  - 传输层协议- TCP：用于保证 数据的可靠传输：不重复/不丢失/有序；
  - 应用层协议- SSL/TLS：用于 加密传输内容，以提高数据安全性

  - 应用层技术- DNS: 域名解析系统，映射解析 IP地址 <--> 域名/主机名
  - 应用层技术- 代理： 数据传输过程中的 中转站，可以分为正向代理/反向代理，如CDN就是一种反向代理
  - 应用层技术- URI / URL:  统一资源标识符/定位符，用于定位 某个主机上的某个特定资源的 具体位置
