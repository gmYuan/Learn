Q1: 什么是 DNS解析

A: DNS 是一个 `应用层协议`, 它可以通过域名 查找 IP地址 + 逆向从 IP地址 反查域名

如: baidu.com => 220.114.23.56 (服务器外网IP地址）80（服务器端口号）


Q2: DNS解析流程

A:

S1 浏览器缓存 => 操作系统 hosts文件缓存 => 路由缓存 => 本地域名服务器 => 根域名服务器 进行递归查询

S2 根域名服务器 => 根域名服务器查询 .com域名服务器的 IP地址 => .baidu.com域名服务器 => 依次类推

[从URL输入到页面展现到底发生什么](https://juejin.im/post/5bf3ad55f265da61682afc9b#heading-10)


Q1: TCP连接为什么要进行 三次握手

A: 为了防止已失效的连接请求报文段突然又传送到了服务端，因而产生错误


Q2: TCP连接流程(3次握手的过程)

A:

SYN: 同步号,表示是否开始建立连接


S1 客户端发送一个 带 SYN=1, ACK = 0, Seq=x 的数据包 到服务器端口

S2 服务器回一个带 SYN=1, ACK=1, Seq=y, ack=x+1 的响应包以示传达确认信息

S3 客户端再回传一个带 ACK=1, Seq=x+1, ack=y+1 的数据包 给服务器


Q3: TCP连接为什么要进行 4次挥手

A: 

S1 A->B 发送关闭请求1，表示A不会再发送数据包;

S2 B->A 发送关闭请求1的响应，表示B收到了A的关闭通知, 否则A会认为B未收到上次的关闭请求1, 从而重新发送;

S3 B->A 发送关闭请求2, 表示B不会再发送数据包了;

S4 A->B 发送关闭请求2的确认, 表示已经接收到

S5 A接收到请求2后，不会立刻关闭TCP连接，防止请求2之前 发送的数据包延迟到达

Q1: HTTP请求报文的结构是什么

A:

S1 请求行（request line） + 请求头（header） + 换行 + 请求体

S2 请求行: `POST  /chapter17/user.html  HTTP/1.1`, 即 请求方法 + URL + 协议版本


Q2: HTTP 请求方法有那些

A:

S1 GET + POST + HEAD + PUT + DELETE + OPTIONS + PATCH + TRACE
