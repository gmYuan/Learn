## <span id="2">2 JS数据类型 </span>

1 Q: JS有哪些数据类型

A: 

S1 JS的数据分类2大类，分别是 基本数据类型 和 引用数据类型

S2.1 基本数据类型包括: number、string、boolean、undefined、null、symbol

S2.2 引用类型: objcet， function、Array都是object类型

S3.1 基本数据类型保存在内存中的栈中，变量直接保存值

S3.2 引用数据类型保存在内存的堆中，变量保存的是 地址的引用(指针)


2 Q: typeof可以返回哪些类型

A:

S1 返回基本数据类型，例外是 null会返回 object

S2 返回引用数据类型，例外是 function会返回 function

S3 关于null返回object的原因，其实是JS的一个久远bug:

> 早期JS用低位存储了变量的类型信息，000 开头代表是对象

> 而 null 表示为全零，所以将它错误的判断为object 

```js
typeof 123          //  "number"
typeof '123'        //  "string"
typeof false        //  "boolean"
typeof undefined    //  "undefined"
typeof Symbol()     // 'symbol'

typeof null         //  "object"


typeof []           //  "object"
typeof {}           //  "object"

let fn = function(){}
typeof fn           //  "function"
```


## <span id="3">3 数据类型转化 </span>

1 Q: JS哪些操作，会有数据类型的 自动转化

A: 

S1 四则运算

S1.1 加法运算: 只要其中一个运算子是 字符串，就会把另一个也转为字符串类型

S1.2 其他运算: 只要其中一个运算子是 数字，就会把另一个也转为数字类型


S2 if运算

S2.1 只有是falsy值，才会显示为假，其余都为真，包括空对象

S2.2 6个falsy值是: '', 0, false, undefined, null,NaN


S3 布尔逻辑运算

S3.1 &&操作符规则: 遇假则返，无假返真

S3.2 ||操作符规则: 遇真则返，无真返假


S4 ==判断

了解即可，一般只使用 ===运算符


```js
// 四则运算举例
1 + '1'            // '11'
2 - '1'            //  1

[1, 2] + [2, 1]    // '1,22,1'
// 对象在转换基本类型时，首先会调用 valueOf，然后调用 toString

// [1, 2].toString() -> '1,2'
// [2, 1].toString() -> '2,1'
// '1,2' + '2,1' = '1,22,1'


'a' + + 'b'      // "aNaN"
// 因为 + 'b' -> NaN, 类似于  + '1' -> 1
```
